1. Укажите применение, характеристики и строение оптоволоконного кабеля. Укажите преимущества и недостатки оптоволоконного кабеля по сравнению с другими видами кабелей.

Применение оптоволоконного кабеля: магистральные линии связи, высокоскоростные сети передачи данных, телекоммуникационные системы, медицинское оборудование, промышленные сети.

Строение:

  • Сердечник (core) из стеклянного или пластикового волокна
  • Оболочка (cladding) с меньшим коэффициентом преломления
  • Защитное покрытие (buffer coating)
  • Армирующие элементы (кевлар, стальные нити)
  • Внешняя защитная оболочка

Характеристики:

  • Пропускная способность до нескольких Тбит/с
  • Дальность передачи без повторителей до 100+ км
  • Диаметр сердечника: 8-62,5 мкм (одномодовое), 50-100 мкм (многомодовое)
  • Рабочие длины волн: 850 нм, 1300 нм, 1550 нм

Преимущества:

  • Высокая скорость передачи данных
  • Невосприимчивость к электромагнитным помехам
  • Большая дальность передачи без потерь
  • Безопасность данных (сложность несанкционированного подключения)
  • Малые габариты и вес

Недостатки:

  • Высокая стоимость оборудования и монтажа
  • Хрупкость волокна, сложность ремонта
  • Необходимость специального оборудования для соединения
  • Ограничения на изгиб кабеля

2. Дайте определение понятиям «интерфейс системного и прикладного программирования (API)», «системный вызов». Опишите принцип работы системных вызовов.

API (Application Programming Interface) — набор готовых классов, процедур, функций, структур и констант, предоставляемых приложением или операционной системой для использования во внешних программных продуктах.

Системный вызов — обращение программы к ядру операционной системы для выполнения привилегированных операций, которые программа не может выполнить самостоятельно.

Принцип работы системных вызовов:

  1. Программа помещает параметры вызова в регистры процессора или стек
  2. Выполняется специальная инструкция (например, int 0x80 в Linux или syscall в современных процессорах)
  3. Процессор переключается в режим ядра и передает управление обработчику системных вызовов ОС
  4. Ядро проверяет параметры и права доступа
  5. Выполняется запрошенная операция
  6. Результат возвращается в пользовательское пространство через регистры или стек
  7. Управление возвращается программе с соответствующим кодом возврата

Примеры системных вызовов: открытие файла (open), чтение/запись (read/write), создание процесса (fork), выделение памяти (mmap).

3. Дайте определение понятию «оперативная память». Перечислите типы оперативной памяти. Охарактеризуйте тайминги оперативной памяти.

Оперативная память (ОЗУ, RAM) — энергозависимая память компьютера, используемая для временного хранения данных и команд, необходимых процессору для выполнения операций.

Типы оперативной памяти:

  • DRAM (Dynamic RAM) — требует постоянного обновления, используется в основной памяти
  • SRAM (Static RAM) — более быстрая, не требует обновления, используется в кэш-памяти
  • SDRAM (Synchronous DRAM) — синхронизирована с системной шиной
  • DDR SDRAM (DDR, DDR2, DDR3, DDR4, DDR5) — передача данных по обоим фронтам тактового сигнала
  • RDRAM (Rambus DRAM) — устаревший высокопроизводительный тип памяти
  • LPDDR — низковольтные версии для мобильных устройств

Тайминги оперативной памяти: задержки между различными операциями с памятью, измеряемые в тактах:

  • CAS Latency (CL) — задержка между запросом столбца и началом передачи данных
  • RAS to CAS Delay (tRCD) — задержка между активацией строки и столбца
  • RAS Precharge (tRP) — время повторной активации банка памяти
  • Row Active Time (tRAS) — минимальное время активности строки
  • Command Rate (CR) — задержка между командой и ее выполнением (обычно 1T или 2T)

Чем меньше значения таймингов, тем быстрее работает память. Например, обозначение 16-18-18-36 означает CL-tRCD-tRP-tRAS соответственно.
